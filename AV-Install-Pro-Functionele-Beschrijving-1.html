Inleiding & Technische Stack

AV Install Pro is een webgebaseerde field service applicatie voor techniekers en projectmanagers die AV-installaties uitvoeren bij klanten. De applicatie draait volledig in de browser en is geoptimaliseerd voor smartphone en tablet.

Kernprincipe: Één HTML-bestand als Single Page Application — gehost op GitHub Pages, gekoppeld aan Supabase als backend. Geen aparte installatie vereist. Werkt op elke moderne browser.
ComponentTechnologiePlatform
FrontendHTML5 / CSS3 / Vanilla JavaScriptBrowser (Chrome, Safari, Edge)
Backend / DatabaseSupabase — PostgreSQL + REST API + AuthCloud (EU-region)
HostingGitHub Pages (stijnborloo/field-assist)CDN — wereldwijd
PDF generatiejsPDF + html2canvasClient-side — geen server nodig
AI integratieClaude Sonnet 4 (Anthropic API)Via directe API-call
SpraakherkenningWeb Speech API (nl-BE)Browser native — Chrome op Android
Foto-compressieCanvas API — max 800px / 65% JPEGClient-side — ~80KB per foto
AuthenticatieSupabase Auth — e-mail + wachtwoordJWT tokens

3.4 Excel Import — 3-stappen wizard

1
Bestand uploaden
Sleep een XLSX of XLS bestand naar het uploadgebied of klik om te bladeren. Template downloadbaar voor de juiste kolomstructuur.
2
Kolommen koppelen
Automatische detectie van kolomnamen. Handmatig aanpasbaar via dropdown per kolom: Locatie, Naam, Artikelnummer, Serienummer, Aantal.
3
Valideren & importeren
Voorbeeldtabel toont de te importeren data. Kies het doelproject en klik "Importeer". Duplicaten worden samengevoegd op basis van serienummer.

3.5 PDF Export

Rapport gegenereerd client-side (jsPDF + html2canvas). Bevat:

  • Projectinformatie, contactgegevens, datum, referentie
  • Zalen met toestellen, serienummers, locatienotities
  • Checklist resultaten (afgevinkt / open)
  • Foto's per zaal (thumbnail grid)
  • Handtekeningen klant en technieker
  • Installatietijden (aankomst, afrondtijd, totaalduur)

Versturen via "Delen met bijlage" → native share sheet → Outlook / Gmail / WhatsApp met PDF als bijlage.

3.6 Planning

  • Kalenderweergave per maand — navigeren via pijlknoppen
  • Gekleurde badges per dag voor interventies en projecten
  • Nieuw planningitem aanmaken: titel, datum, technieker, project
  • Items bekijken en verwijderen

3.7 Interventies (admin)

  • Volledige lijst van alle interventies met status-badge teller in de zijbalk
  • Filteren op status en prioriteit
  • Nieuwe interventie aanmaken via modal: titel, beschrijving, prioriteit, project, adres, technieker, datum
  • GPS-locatie ophalen voor adresveld
  • Status manueel bijwerken
  • Interventie verwijderen
  • Badge in zijbalk toont aantal open interventies

3.8 Teambeheer

  • Overzicht van alle geregistreerde gebruikers uit de profiles-tabel
  • Naam en initialen bewerken
  • Rol aanpassen: admin / planner / technieker
  • Nieuwe gebruikers worden automatisch aangemaakt bij registratie (trigger)
  • Gebruiker verwijderen (soft delete via Supabase)

Projectflow Technieker

De technieker volgt een gestructureerde 5-stappenstroom per project. Elke stap registreert automatisch tijdstempels en synchroniseert naar Supabase.

1
🧭 Vertrek naar klant
Knop "Vertrek naar klant" in de Info-tab. Opent navigatiemenu met Google Maps, Waze of Apple Maps. Interventies worden automatisch bijgewerkt naar "Onderweg".
2
📍 Aankomst registreren
Knop "Aankomst registreren" verschijnt na vertrek. Registreert exacte aankomsttijd, start de project-timer automatisch. Interventies → "Ter plaatse".
3
🔧 Installatie uitvoeren
Per zaal: status bijwerken, timer start/stopt automatisch, foto's nemen, toestellen bewerken, notities inspreken via 🎤. Checklist afvinken met live voortgangsmeter.
4
✍️ Handtekening verzamelen
Klant tekent op scherm, daarna technieker. Na beide handtekeningen → automatisch naar Versturen. Opgeslagen als PNG in Supabase.
5
🏁 Afronden & Versturen
Controlelijst toont status van alle onderdelen. Bevestigen stopt timer, zet project op "Afgerond". PDF wordt gedeeld via native share (Outlook, Gmail, WhatsApp).

Interventies

Interventies zijn zelfstandige tussenkomsten bij een klant. Ze kunnen gekoppeld zijn aan een project maar ook volledig los staan.

6.1 Statusflow

⚪ Open → 🔵 Toegewezen → 🟠 Onderweg → 🟣 Ter plaatse → ✅ Afgerond

6.2 Prioriteiten

🔴 Urgent 🟠 Hoog 🔵 Normaal ⚪ Laag

6.3 Interventie detail — snelle acties

  • 📷 Foto toevoegen (type kiezen: Voor / Na / Defect / Bekabeling / Serienummer / Algemeen)
  • 📞 Bellen naar klant (deeplink naar telefoon)
  • 📁 Project openen (als gekoppeld)
  • 📄 PDF rapport openen
  • 🤖 AI verslag genereren (Claude Sonnet)
  • 🎤 Notitie inspreken (Web Speech API)
  • 🧭 Navigeren naar adres

6.4 Interventierapport

Standalone PDF-rapport per interventie met 3-tab flow:

  • Verslag — samenvatting, checklist, foto-overzicht, download
  • Aftekenen — klant + technieker digitale handtekening
  • Versturen — e-mailadres, bericht, "Delen met bijlage" knop
AI rapportgeneratie: Claude analyseert alle interventiedata en schrijft automatisch een professioneel verslag in drie secties: Situatiebeschrijving, Uitgevoerde werkzaamheden, Resultaat & opmerkingen. De technieker kan het resultaat gebruiken, opnieuw genereren of bewerken.

Tijdregistratie

8.1 Project-timer

TriggerActie
"Aankomst registreren" knopTimer start — aankomsttijd geregistreerd
Status interventie → "Ter plaatse"Timer start automatisch
"Project afronden" bevestigenTimer stopt — afrondtijd geregistreerd
Manueel via Info-tab knopStart / Pauzeer / Hervat / Afronden

8.2 Zaal-timer

TriggerActie
Status → "In uitvoering"Zaal-timer start automatisch
Status → "Afgerond"Zaal-timer stopt + opgeslagen in room_meta
Status → "Gepland"Zaal-timer pauzeert
Manuele knopStart / Pauzeer ongeacht status

8.3 Synchronisatie naar Supabase

  • timer_elapsed — totale verstreken tijd in seconden
  • timer_arrival — aankomsttijdstempel (ISO 8601)
  • timer_finish — afrondtijdstempel
  • timer_sessions — array van sessies met start/end/duration
  • room_meta — JSON per zaal: status + timer + notitie
Admin inzicht: De admin ziet in de Zalen-tab per zaal de installatietijd, status, notitie van de technieker en foto-teller — real-time gesynchroniseerd vanuit het veld.

9.2 Automatische profielaanmaak

Bij registratie van een nieuwe gebruiker maakt een PostgreSQL-trigger on_auth_user_created automatisch een profiel aan. Rol is standaard "technieker" — aanpasbaar via SQL of admin-interface.

9.3 RLS — Row Level Security

Alle tabellen hebben RLS ingeschakeld. Ingelogde gebruikers hebben lees- en schrijftoegang. De fotos- en profiles-tabel zijn leesbaar voor alle ingelogde gebruikers.

9.4 Offline gedrag

  • Datawijzigingen worden eerst lokaal toegepast voor directe UI-respons
  • Supabase-synchronisatie verloopt asynchroon op de achtergrond
  • Foto's en timers worden extra opgeslagen in localStorage als backup
  • Bij herladen worden localStorage-backups hersteld
  • Bij netwerkfout: melding "Lokaal opgeslagen" — data gaat niet verloren
10

UX Details & Bijlagen

10.1 Browser back-knop

De applicatie onderschept de browser back-knop via de History API. Elke overlay pusht een state — back sluit de overlay in plaats van de pagina te verlaten.

  • Interventiedetail → back → interventies-overzicht
  • Projectenlijst → back → meer/vandaag
  • Projectdetail → back → projectenlijst
  • Rapport, navigatie, horloge → back → sluit modal

10.2 Smartwatch notificaties

Via Web Notifications API verschijnen meldingen op gekoppelde WearOS/Apple Watch horloges (via telefoonnotificaties). Configureerbaar per type: nieuwe interventie, urgent, statuswijziging, checklist klaar, handtekening.

10.3 Foto-compressie

  • Max breedte én hoogte: 800px
  • JPEG-kwaliteit: 65%
  • Typisch: 4MB origineel → 80-150KB opgeslagen (25-50x kleiner)
  • Canvas-geheugen vrijgegeven na compressie

10.4 Navigatie naar klant

  • Google Maps — opens.google.com/maps
  • Waze — waze:// deep link (opent app direct)
  • Apple Maps — maps:// deep link
